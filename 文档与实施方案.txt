# WanyueKJ/education-online 项目参考下的教务系统需求文档与实施方案
项目名称：在线教务管理系统（Education Administration System）
参考项目：WanyueKJ/education-online（主仓库 Web + 后台管理，PHP/ThinkPHP + PhalApi + MySQL） + wanyue_online_education小程序（Uni-app 多端前端）
参考项目核心价值：成熟的在线教育全栈方案（知识付费、直播、题库/考试、支付、多端支持、角色权限、统计），100% 开源，可商用（标注来源）。我们以此为蓝本，完全重构为 Django + Vue + Uni-app 技术栈，满足用户指定的前后端分离、Docker 部署、Redis 缓存、定时备份等需求。
当前时间：2026 年 4 月（参考项目最后更新 2025 年 3 月，仍为活跃成熟项目）

1. 整体思路与架构设计
1.1 架构概览（前后端分离 + 多端统一）
* 前端：
    * Web 用户界面（学生/老师/游客切换）：Vue 3 + Vite + Pinia + Vue Router + Element Plus / Vant（SPA）
    * 后台管理界面（仅 Admin）：Vue 3 独立 Admin 项目（或同一仓库子目录）
    * App + 微信小程序 + H5：Uni-app（基于 Vue）（一套代码编译多端，参考 Wanyue Uni-app 仓库）
* 后端：Django 5.x + Django REST Framework (DRF) + drf-spectacular（自动 API 文档）
* 中间件：
    * MySQL 8.x（主存储）
    * Redis 7.x（缓存、Session、限流、Celery Broker）
* 部署：Docker + Docker Compose（全容器化，一键部署）
* 其他：
    * 认证：JWT（DRF + djangorestframework-simplejwt） + 手机号/微信/QQ 一键登录
    * 任务队列：Celery + Redis（异步、定时任务）
    * 地图/图表：ECharts + Gaode Map / Baidu Map API（用户来源统计、柱状图）
    * 文件存储：本地 + 可扩展至阿里云 OSS / 七牛云（参考 Wanyue SDK）
    * 支付/第三方登录：微信/QQ/手机号（接入官方 SDK + 短信服务）
架构图（文字版）：

用户端 (Web / Uni-app)  <--- REST API + WebSocket ---> Django DRF Backend
    │
    ├─ Vue / Uni-app (前端)
    ├─ MySQL (数据)
    ├─ Redis (缓存 + 队列)
    └─ Docker (Nginx + uWSGI/Gunicorn + Celery Beat)
后台管理 (Vue Admin) ────────────────────────────────► 同上

为什么选择此栈：
* Django：快速开发 admin、ORM、权限系统（参考 Wanyue 后台权限管理）。
* Vue + Uni-app：前端体验现代，Uni-app 完美复用 Wanyue 多端逻辑。
* Docker：满足“后期部署 + 定时备份”要求。
* Redis：高频数据缓存、错题本实时记录、会话管理（比 Wanyue 原生缓存更高效）。
1.2 环境配置（本地开发 + 生产）
本地开发环境
1. 安装 Docker Desktop（推荐）或手动安装 Python 3.12 + Node 20。
2. 克隆仓库结构（建议）：education-admin-system/

4. ├── frontend-web/     # Vue Web + Admin
5. ├── frontend-uniapp/  # Uni-app 多端

32. 定时备份（生产必备）：
    * 使用 Celery Beat + cron 或 Docker 中的 crontab。
    * 每日 02:00 执行 mysqldump 备份到 /backup 卷（可挂载宿主机或对象存储）。
生产部署（推荐阿里云/腾讯云）
* 使用 Docker Swarm 或 Kubernetes（后期扩展）。
* Nginx 反代 + HTTPS（Let’s Encrypt）。
* 数据库备份自动上传至 OSS。
* 监控：Prometheus + Grafana（可选）。

2. 详细需求文档
2.1 用户角色与权限（参考 Wanyue 多端 + 后台权限）
角色	描述	权限控制方式
游客	免登录查看课程大纲、首页、活动	管理员后台开关模块
学生	刷题、作业、错题本、评价查看	普通用户
老师	查看名下学员进度、留言评价	后台赋权为“管理员子角色”
Admin	超级管理员（唯一登录账号）	后台管理所有账户、权限、模块

登录/注册（必填项）：
* 支持：手机号 + 短信验证码、QQ、微信（一键授权）
* 注册表单：手机号/QQ/微信（三选一）、密码、确认密码、昵称、籍贯（必填）
* 一键登录：手机号授权（参考微信/支付宝小程序能力）
* 账号设置：修改头像、昵称、籍贯；密码重置需验证码
2.2 核心功能模块
用户访问界面（Web + App + 小程序）：
* 首页：轮播图、搜索、课程大纲、往期优秀学员展示、高频面试题展示、就业城市分布图、最高薪资前十信息条（滚动播报）、师资力量展示。
* 活动中心：发起活动（表单提交 → 存 DB），游客可查看。
* 题库与作业：
    * 题库支持文字、图片、语音。
    * 分为三个阶段（可切换，不同表数据或同一表 stage 字段）。
    * 按分类划分（科目/难度/类型）。
    * 作业 = 随机抽取题库题目。
* 错题本：刷题错误自动记录；用户不可手动更改状态；答对后提示“是否移出错题本”。
* 后台地图（学生/老师可见）：默认校址红标；用户来源城市分布（每期学员各省/市总人数 + 往期对比）；柱状图展示。
* 评价：老师（管理员身份）查看学员刷题进度 → 留言评价。
后台管理界面（仅 Admin）：
* 单账号登录（demo/admin）。
* 账户管理：增删改查、给老师赋“管理员”权限。
* 权限控制：所有用户访问界面的模块开关。
* 数据统计、导出（参考 Wanyue PHPExcel）。
* 题库/活动/课程全量 CRUD。
非功能需求：
* 响应式 + 多端适配（Uni-app 负责）。
* 安全性：JWT + 验证码防刷、Redis 限流、SQL 注入防护。
* 性能：Redis 缓存高频数据（题库、统计）。
* 可扩展：支持后期直播、支付（直接复用 Wanyue SDK 思路）。
