# 在线教务管理系统 - 需求文档与实施方案

## 项目概述

**项目名称**: 在线教务管理系统 (Education Administration System)  
**参考项目**: WanyueKJ/education-online (GitHub 1k+ stars, 开源可商用)  
**技术栈重构**: Django + Vue + Uni-app (替代原 PHP + ThinkPHP)  
**编制日期**: 2026年4月21日

---

## 一、参考项目分析

### 1.1 WanyueKJ/education-online 项目信息

- **主仓库**: https://github.com/WanyueKJ/education-online
- **技术栈**: PHP + ThinkPHP + PhalApi + MySQL
- **小程序/App**: https://github.com/WanyueKJ/wanyue_online_education_uniapp
- **Stars**: 1000+ (成熟项目,持续维护至2025年)
- **协议**: 100%开源,可商用(建议标注来源)
- **核心价值**: 
  - 完整的在线教育全栈方案
  - 知识付费、直播、题库/考试、支付、多端支持
  - 角色权限、统计分析等成熟功能

### 1.2 为什么选择重构为 Django + Vue

1. **后端**: Django 提供快速开发、强大的 ORM、内置 Admin、完善的权限系统
2. **前端**: Vue 3 生态成熟,Uni-app 完美支持多端(微信小程序/App/H5)
3. **部署**: Docker 容器化,便于后期运维和扩展
4. **性能**: Redis 缓存高频数据,比原生 PHP 缓存更高效

## 二、系统架构设计

### 2.1 整体架构(前后端分离 + 多端统一)

```
┌─────────────────────────────────────────────────────────┐
│  用户端 (Web / App / 小程序)                              │
│  - Vue 3 Web (学生/老师/游客切换)                         │
│  - Uni-app (微信小程序 + App + H5)                       │
└────────────────┬────────────────────────────────────────┘
                 │ REST API + WebSocket
                 ↓
┌─────────────────────────────────────────────────────────┐
│  后端服务 (Django + DRF)                                 │
│  - Django REST Framework (API)                          │
│  - JWT 认证 (djangorestframework-simplejwt)             │
│  - Celery (异步任务 + 定时备份)                          │
└────────────────┬────────────────────────────────────────┘
                 │
    ┌────────────┼────────────┐
    ↓            ↓            ↓
┌────────┐  ┌────────┐  ┌────────┐
│ MySQL  │  │ Redis  │  │ Nginx  │
│  8.x   │  │  7.x   │  │ (反代) │
└────────┘  └────────┘  └────────┘
```

### 2.2 技术栈明细

**后端**:
- Django 5.x + Django REST Framework (DRF)
- djangorestframework-simplejwt (JWT认证)
- drf-spectacular (自动生成API文档)
- Celery + Redis (异步任务队列)
- mysqlclient (MySQL驱动)

**前端**:
- Vue 3 + Vite + Pinia + Vue Router
- Element Plus (PC端) / Vant (移动端)
- Axios (HTTP客户端)
- ECharts (数据可视化)

**多端**:
- Uni-app (基于Vue,一套代码编译多端)
- 支持: 微信小程序 + App + H5

**中间件**:
- MySQL 8.x (主存储)
- Redis 7.x (缓存 + Session + Celery Broker)

**部署**:
- Docker + Docker Compose (一键部署)
- Nginx (反向代理 + 静态文件)
- Gunicorn / uWSGI (WSGI服务器)

## 三、核心功能需求

### 3.1 用户角色与权限

| 角色 | 描述 | 权限控制方式 |
|------|------|--------------|
| 游客 | 免登录查看课程大纲、首页、活动 | 后台开关控制模块可见性 |
| 学生 | 刷题、作业、错题本、查看评价 | 普通用户权限 |
| 老师 | 查看名下学员进度、留言评价 | 后台赋权为"管理员子角色" |
| Admin | 超级管理员(唯一登录账号) | 后台管理所有账户、权限、模块 |

### 3.2 登录/注册功能

**支持方式**:
- 手机号 + 短信验证码
- QQ 一键登录
- 微信一键授权登录

**注册表单(必填项)**:
- 登录方式: 手机号/QQ/微信 (三选一)
- 密码 + 确认密码
- 昵称
- 籍贯

**账号设置**:
- 修改头像、昵称、籍贯
- 密码重置(需验证码)

### 3.3 用户访问界面功能

#### 3.3.1 首页
- 轮播图
- 搜索功能
- 课程大纲展示
- 往期优秀学员展示
- 高频面试题展示
- 就业城市分布图
- 最高薪资前十信息条(滚动播报)
- 师资力量展示

#### 3.3.2 活动中心
- 发起活动(表单提交 → 存入数据库)
- 游客可查看活动列表

#### 3.3.3 题库与作业

**题库特性**:
- 支持题目类型: 文字、图片、语音
- 分为三个阶段(可切换,不同表数据或同表 stage 字段)
- 按分类划分(科目/难度/类型)
- 作业 = 随机抽取题库题目

**题目分类**:
- 科目分类
- 难度分类
- 题型分类

#### 3.3.4 错题本
- 刷题错误自动记录
- 用户不可手动更改状态
- 答对后提示"是否移出错题本"

#### 3.3.5 后台地图(学生/老师可见)
- 默认校址红标展示
- 用户来源城市分布
- 统计每期学员各省/市总人数
- 往期总人数、城市分布
- 分布图下方以柱状图表示

#### 3.3.6 评价功能
- 老师(管理员身份)查看学员刷题进度
- 老师对学员留言评价

### 3.4 后台管理界面(仅 Admin)

**核心功能**:
- 单账号登录(demo/admin)
- 账户管理: 增删改查、给老师赋"管理员"权限
- 权限控制: 所有用户访问界面的模块开关
- 数据统计、导出(参考 Wanyue PHPExcel)
- 题库/活动/课程全量 CRUD

## 四、数据库设计

### 4.1 核心模型(Django ORM)

```python
# 用户模型
class User(AbstractUser):
    phone = models.CharField(max_length=11, unique=True, null=True, blank=True)
    qq = models.CharField(max_length=20, null=True, blank=True)
    wechat_openid = models.CharField(max_length=100, null=True, blank=True)
    nickname = models.CharField(max_length=50)
    hometown = models.CharField(max_length=100)  # 籍贯
    avatar = models.ImageField(upload_to='avatars/', null=True, blank=True)
    role = models.CharField(max_length=20, choices=[
        ('guest', '游客'),
        ('student', '学生'),
        ('teacher', '老师'),
        ('admin', '管理员')
    ], default='student')
    created_at = models.DateTimeField(auto_now_add=True)

# 题库模型
class Question(models.Model):
    stage = models.IntegerField(choices=[(1,'阶段1'), (2,'阶段2'), (3,'阶段3')])
    category = models.CharField(max_length=50)  # 分类
    subject = models.CharField(max_length=50)  # 科目
    difficulty = models.CharField(max_length=20)  # 难度
    content = models.TextField()  # 题目内容
    image = models.ImageField(upload_to='questions/', null=True, blank=True)
    voice = models.FileField(upload_to='questions/audio/', null=True, blank=True)
    answer = models.TextField()  # 答案
    created_at = models.DateTimeField(auto_now_add=True)

# 错题本模型
class WrongQuestion(models.Model):
    user = models.ForeignKey(User, on_delete=models.CASCADE)
    question = models.ForeignKey(Question, on_delete=models.CASCADE)
    is_correct = models.BooleanField(default=False)  # 答对后可移除
    created_at = models.DateTimeField(auto_now_add=True)

# 活动模型
class Activity(models.Model):
    creator = models.ForeignKey(User, on_delete=models.CASCADE)
    title = models.CharField(max_length=200)
    description = models.TextField()
    form_data = models.JSONField()  # 表单数据
    created_at = models.DateTimeField(auto_now_add=True)

# 评价模型
class Evaluation(models.Model):
    teacher = models.ForeignKey(User, on_delete=models.CASCADE, related_name='given_evaluations')
    student = models.ForeignKey(User, on_delete=models.CASCADE, related_name='received_evaluations')
    content = models.TextField()
    created_at = models.DateTimeField(auto_now_add=True)

# 统计模型(学员分布)
class StudentStats(models.Model):
    period = models.CharField(max_length=50)  # 期数
    province = models.CharField(max_length=50)
    city = models.CharField(max_length=50)
    count = models.IntegerField(default=0)
    created_at = models.DateTimeField(auto_now_add=True)
```

## 五、实施方案(分阶段执行计划)

### 阶段 1: 项目初始化 (1-2周)

**任务清单**:
1. 创建 Git 仓库,初始化目录结构
2. Backend: `django-admin startproject backend`
   - 安装依赖: DRF, simplejwt, celery, redis, mysqlclient, drf-spectacular
   - 配置 settings.py: MySQL, Redis, CORS, JWT
3. Frontend: 
   - Vue 3 Web: `npm create vite@latest frontend-web -- --template vue`
   - Uni-app: `npx degit dcloudio/uni-preset-vue frontend-uniapp`
4. Docker Compose 一键启动(`docker compose up -d`)

**目录结构**:
```
education-admin-system/
├── backend/          # Django 项目
├── frontend-web/     # Vue Web + Admin
├── frontend-uniapp/  # Uni-app 多端
├── docker-compose.yml
├── .env.example
└── docs/             # 本文档
```

### 阶段 2: 数据库设计与核心模型 (1周)

**任务清单**:
1. 使用 Django ORM 创建上述模型
2. 运行 `makemigrations` & `migrate`
3. 导入初始题库数据(Excel → Django admin)
4. 创建超级用户: `python manage.py createsuperuser`

### 阶段 3: API 与前端开发 (4-6周)

**后端 API (DRF)**:
- User: 注册/登录/一键登录
- Question: 分页、随机、分类、阶段切换
- WrongBook: 增删查
- Activity: 创建、列表
- Stats: 地图数据 JSON
- Evaluation: 老师评价学员

**前端开发**:
- Web Vue: Pinia 状态管理, Axios 调用 API
- Uni-app: 复用 Wanyue Uni-app 的登录、题库、Socket 逻辑
- 实现游客模式(Token 为 None 或 guest flag)
- 地图: 前端 ECharts + 后端聚合用户籍贯/来源数据

### 阶段 4: 高级功能与测试 (2周)

**任务清单**:
1. 错题本逻辑、老师评价、后台权限(Django Guardian 或自定义)
2. 定时备份 Celery Task:
```python
@periodic_task(run_every=crontab(hour=2, minute=0))
def backup_database():
    # 调用 mysqldump 或 django-dbbackup
```
3. 单元测试 + Postman 集合测试所有接口

### 阶段 5: 部署与上线 (1周)

**任务清单**:
1. 完善 `docker-compose.prod.yml`
2. 部署到服务器,配置域名 + HTTPS
3. 监控日志(ELK 或简单文件日志)
4. 提供运维文档(备份恢复、一键部署脚本)

## 六、环境配置

### 6.1 本地开发环境

**前置要求**:
- Docker Desktop (推荐) 或 Python 3.12 + Node 20
- Git

**快速启动**:
```bash
# 1. 克隆仓库
git clone <repo-url> education-admin-system
cd education-admin-system

# 2. 配置环境变量
cp .env.example .env
# 编辑 .env 设置数据库密码等

# 3. 启动所有服务
docker compose up -d

# 4. 初始化数据库
docker compose exec backend python manage.py migrate
docker compose exec backend python manage.py createsuperuser

# 5. 访问
# - 前端: http://localhost:3000
# - 后台: http://localhost:8000/admin
# - API文档: http://localhost:8000/api/docs
```

### 6.2 Docker Compose 配置核心片段

```yaml
services:
  db:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: ${DB_ROOT_PASS}
      MYSQL_DATABASE: education
    volumes:
      - mysql_data:/var/lib/mysql
      - ./backup:/backup  # 定时备份目录
  
  redis:
    image: redis:7-alpine
  
  backend:
    build: ./backend
    command: gunicorn --workers 4 backend.wsgi:application
    depends_on: [db, redis]
    env_file: .env
  
  frontend:
    build: ./frontend-web
    depends_on: [backend]

volumes:
  mysql_data:
```

### 6.3 定时备份(生产必备)

**方式 1: Celery Beat**
```python
# backend/tasks.py
from celery import shared_task
from celery.schedules import crontab
import subprocess

@shared_task
def backup_database():
    subprocess.run([
        'mysqldump',
        '-h', 'db',
        '-u', 'root',
        '-p' + os.getenv('DB_ROOT_PASS'),
        'education',
        '>', f'/backup/backup_{datetime.now().strftime("%Y%m%d")}.sql'
    ], shell=True)
```

**方式 2: Docker crontab**
```dockerfile
# 在 backend Dockerfile 中添加
RUN echo "0 2 * * * /app/backup.sh" | crontab -
```

- **短信验证码**: 接入阿里云/腾讯云短信服务
