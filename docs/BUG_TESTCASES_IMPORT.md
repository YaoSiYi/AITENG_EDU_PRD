# Bug 测试用例导入文档

## 概述

本文档记录了将项目中的 Bug 和优化需求转换为测试用例并导入到测试用例管理系统的过程。

---

## Bug 和需求列表

### Bug 记录

#### BUG-007: 测试用例管理页为空
- **严重程度**: 高
- **优先级**: 紧急
- **状态**: 已关闭
- **问题描述**: 访问测试用例管理页面时页面显示为空白，无法正常加载用例列表
- **根本原因**: 端口冲突、缺失导入、缺失函数等问题
- **修复方案**: 修复了端口冲突(3000->3001)、添加缺失的导入语句、补充缺失的函数实现

#### BUG-006: 测试用例状态缺少"已关闭"
- **严重程度**: 中
- **优先级**: 高
- **状态**: 已关闭
- **问题描述**: 测试用例的状态字段缺少"已关闭"选项，无法将用例标记为已关闭状态
- **修复方案**: 新增 closed 状态，同步到筛选、编辑弹窗、列表快捷修改、标签显示样式

#### BUG-005: 个人中心编辑资料按钮无效
- **严重程度**: 中
- **优先级**: 高
- **状态**: 已关闭
- **问题描述**: 点击个人中心的"编辑资料"按钮后无任何反应，无法打开编辑对话框
- **修复方案**: 修复了按钮点击事件绑定问题

### 优化记录

#### 优化-001: 时间格式不统一
- **严重程度**: 低
- **优先级**: 中
- **状态**: 已关闭
- **问题描述**: 系统中不同模块的时间显示格式不一致，影响用户体验
- **优化方案**: 创建全局时间格式化工具 utils/datetime.js，统一所有模块时间显示格式

#### 优化-002: 冒烟用例字段类型不合理
- **严重程度**: 中
- **优先级**: 高
- **状态**: 已关闭
- **问题描述**: 冒烟用例使用布尔型字段，无法支持多阶段测试流程，扩展性差
- **优化方案**: 将布尔型改为枚举型 stage 字段，支持 smoke/pre_prod/production 三个阶段

#### 优化-003: 生产环境阶段颜色不够醒目
- **严重程度**: 低
- **优先级**: 中
- **状态**: 已关闭
- **问题描述**: 生产环境阶段使用绿色标签，不够醒目，无法突出生产环境的重要性和风险性
- **优化方案**: 将生产环境阶段颜色从绿色(success)改为红色(danger)

#### 优化-004: 已关闭状态颜色不够友好
- **严重程度**: 低
- **优先级**: 低
- **状态**: 已关闭
- **问题描述**: 已关闭状态使用红色标签，给人负面感觉，不够友好
- **优化方案**: 将已关闭状态颜色从红色(danger)改为蓝色(primary)

### 需求记录

#### 需求-001: 已关闭和已废弃用例应限制编辑
- **严重程度**: 高
- **优先级**: 高
- **状态**: 已关闭
- **需求描述**: 已关闭和已废弃状态的用例仍可以编辑所有字段，可能导致数据混乱
- **实现方案**: 已关闭状态整行蓝色高亮且锁定编辑，已废弃状态整行灰色显示且锁定编辑

#### 需求-002: 管理后台无法查看用户密码
- **严重程度**: 中
- **优先级**: 高
- **状态**: 已关闭
- **需求描述**: 管理后台只显示哈希密码，管理员无法查看和重置用户密码，不便于用户支持
- **实现方案**: 新增 plain_password 字段存储明文密码，管理后台可查看和修改

#### 需求-003: 前台界面缺少登出功能
- **严重程度**: 高
- **优先级**: 紧急
- **状态**: 已关闭
- **需求描述**: 前台界面没有登出按钮，用户无法退出登录，只能关闭浏览器或清除缓存
- **实现方案**: 个人中心左侧菜单新增退出账号，主页导航栏新增下拉菜单含退出选项

#### 需求-004: 个人中心无法修改密码
- **严重程度**: 高
- **优先级**: 高
- **状态**: 已关闭
- **需求描述**: 个人中心设置页面没有密码修改功能，用户无法自行修改登录密码
- **实现方案**: 新增密码修改模块，支持原始密码验证、新密码设置(6-20位)、确认密码验证、二次确认操作

#### 需求-005: 缺少用例复用功能
- **严重程度**: 中
- **优先级**: 高
- **状态**: 已关闭
- **需求描述**: 创建相似用例时需要重复输入大量相同信息，效率低下
- **实现方案**: 新建用例时新增复用按钮，输入用例ID可快速复制信息，标题自动添加(复制)后缀

## 导入步骤

### 方式一：通过 Web 界面导入

1. **访问测试用例管理页面**
   ```
   http://localhost:3001/testcases
   ```

2. **点击导入按钮**
   - 在页面顶部找到"导入"按钮
   - 点击打开导入对话框

3. **选择 CSV 文件**
   - 点击"选择文件"按钮
   - 选择 `docs/bug_testcases_import.csv`
   - 或直接拖拽文件到上传区域

4. **确认导入**
   - 点击"确定导入"按钮
   - 等待导入完成
   - 查看导入结果提示

5. **验证导入结果**
   - 刷新页面
   - 查看用例列表
   - 验证 12 条用例是否全部导入成功

### 方式二：通过 API 导入

```bash
# 使用 curl 命令导入
curl -X POST http://localhost:8000/api/testcases/import/ \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -F "file=@docs/bug_testcases_import.csv"
```

## 注意事项

### 1. 字段映射

CSV 文件中的字段名必须与系统字段完全匹配：
- ✅ 正确: `产品`, `模块`, `子模块`
- ❌ 错误: `product`, `module`, `sub_module`

### 2. 枚举值

枚举字段必须使用英文值：
- 严重程度: `high`, `medium`, `low`
- 优先级: `critical`, `high`, `medium`, `low`
- 状态: `draft`, `active`, `deprecated`, `closed`
- 阶段: `smoke`, `pre_prod`, `production`

### 3. 编码格式

CSV 文件必须使用 UTF-8 编码，否则中文会乱码。

### 4. 必填字段

以下字段为必填：
- 产品
- 模块
- 标题

### 5. 字段长度限制

- 标题: 最大 200 字符
- 描述: 无限制
- 其他文本字段: 最大 500 字符
