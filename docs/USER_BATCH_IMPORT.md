# 用户批量导入功能文档

## 更新日期：2026-04-24

---

## 🎯 功能概述

为后台管理系统的用户管理页面新增批量导入功能，支持通过 CSV 文件批量导入用户数据。

---

## ✨ 新增功能

### 1. 批量导入按钮

在用户管理列表页面右上角新增"批量导入"按钮，点击后进入批量导入页面。

### 2. CSV 模板下载

提供 CSV 模板下载功能，用户可以下载标准模板，按照模板格式填写数据。

### 3. CSV 文件上传

支持上传 CSV 格式文件，系统会自动解析并导入用户数据。

### 4. 导入结果反馈

导入完成后显示详细的导入结果，包括成功数量、失败数量和错误信息。

---

## 📋 CSV 文件格式

### 必填字段

| 字段名 | 说明 | 示例 |
|--------|------|------|
| username | 用户名（唯一） | student001 |
| password | 密码 | password123 |

### 可选字段

| 字段名 | 说明 | 可选值 | 默认值 | 示例 |
|--------|------|--------|--------|------|
| nickname | 昵称 | - | 空 | 张三 |
| phone | 手机号 | - | 空 | 13800138000 |
| qq | QQ号 | - | 空 | 123456789 |
| hometown | 籍贯 | - | 空 | 广东省深圳市 |
| password_hint | 密码提示 | - | 空 | 我的生日 |
| period | 期数 | - | 空 | 2024-01-01期学员 |
| study_status | 学习状态 | studying, graduated | studying | studying |
| role | 角色 | student, teacher, admin | student | student |
| is_active | 是否激活 | True, False | True | True |

### CSV 文件示例

```csv
username,password,nickname,phone,qq,hometown,password_hint,period,study_status,role,is_active
student001,password123,张三,13800138000,123456789,广东省深圳市,我的生日,2024-01-01期学员,studying,student,True
student002,password456,李四,13800138001,987654321,北京市北京市,我的宠物名,2024-01-01期学员,graduated,student,True
student003,password789,王五,13800138002,555666777,上海市上海市,我的母校,2024-03-15期学员,studying,student,True
```

## 🚀 使用步骤

### 步骤 1: 下载 CSV 模板

1. 访问管理后台：http://localhost:8000/admin/
2. 登录管理员账号（admin / admin123）
3. 进入"用户管理" → "用户"
4. 点击右上角"批量导入"按钮
5. 在导入页面点击"📥 下载 CSV 模板"按钮
6. 保存模板文件到本地

### 步骤 2: 填写用户数据

1. 使用 Excel 或文本编辑器打开模板文件
2. 按照模板格式填写用户数据
3. 确保必填字段（username、password）不为空
4. 确保用户名唯一，不与现有用户重复
5. 保存文件为 CSV 格式（UTF-8 编码）

### 步骤 3: 上传 CSV 文件

1. 在批量导入页面点击"选择 CSV 文件"
2. 选择填写好的 CSV 文件
3. 点击"🚀 开始导入"按钮
4. 确认导入操作

### 步骤 4: 查看导入结果

1. 导入完成后，页面会显示导入结果
2. 成功导入的用户数量
3. 失败导入的用户数量和错误信息
4. 返回用户列表查看导入的用户

---

## 📊 导入规则

### 1. 用户名唯一性检查

- 如果用户名已存在，该行将被跳过
- 错误信息：`第 X 行：用户名 xxx 已存在`

### 2. 必填字段验证

- username 和 password 不能为空
- 错误信息：`第 X 行：用户名不能为空` 或 `第 X 行：密码不能为空`

### 3. 数据格式验证

- phone 字段如果为空，将设置为 NULL
- study_status 默认为 `studying`
- role 默认为 `student`
- is_active 默认为 `True`

### 4. 密码处理

- 密码会自动加密存储
- plain_password 字段会保存明文密码（用于管理后台显示）

## ⚠️ 注意事项

### 1. 文件编码

- **必须使用 UTF-8 编码**
- 模板文件已包含 BOM 标记，支持 Excel 正确显示中文
- 如果使用文本编辑器，请确保保存为 UTF-8 编码

### 2. 数据安全

- 导入操作不可撤销
- 建议先小批量测试
- 导入前备份现有数据
- 不要导入敏感信息

### 3. 性能考虑

- 单次导入建议不超过 1000 条数据
- 大批量导入建议分批进行
- 导入过程中不要关闭页面

### 4. 错误处理

- 如果某行导入失败，不影响其他行
- 错误信息最多显示前 10 条
- 详细错误信息会在页面上显示
