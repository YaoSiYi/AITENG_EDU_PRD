# 用户批量导出功能文档

## 更新日期：2026-04-24

---

## 🎯 功能概述

为后台管理系统的用户管理页面新增批量导出功能，支持导出所选用户的详细数据到 CSV 文件。

---

## ✨ 新增功能

### 1. 批量导出动作

在用户管理列表页面的"动作"下拉菜单中新增"导出所选用户"选项。

### 2. 选择导出用户

- 可以勾选单个或多个用户
- 可以使用"全选"功能选择所有用户
- 支持跨页选择

### 3. 导出 CSV 文件

- 自动生成带时间戳的文件名
- 包含用户的所有详细信息
- 支持 Excel 正确显示中文（UTF-8-sig 编码）

### 4. 导出结果反馈

导出完成后显示成功消息，提示导出的用户数量。

## 🚀 使用步骤

### 步骤 1: 访问用户管理页面

1. 访问管理后台：http://localhost:8000/admin/
2. 登录管理员账号（admin / admin123）
3. 进入"用户管理" → "用户"

### 步骤 2: 选择要导出的用户

**方式一：选择单个或多个用户**
1. 勾选要导出的用户（点击每行前面的复选框）
2. 可以选择多个用户

**方式二：全选所有用户**
1. 点击表头的复选框，全选当前页的所有用户
2. 如果要选择所有页的用户，点击"选择所有 X 个用户"链接

### 步骤 3: 执行导出操作

1. 在页面上方的"动作"下拉菜单中选择"导出所选用户"
2. 点击"执行"按钮
3. 浏览器会自动下载 CSV 文件

### 步骤 4: 查看导出文件

1. 找到下载的 CSV 文件（文件名格式：`users_export_YYYYMMDD_HHMMSS.csv`）
2. 使用 Excel 或文本编辑器打开文件
3. 查看导出的用户数据

## 🔧 技术实现

### 1. 后端实现

**文件路径**：`backend/apps/users/admin.py`

**核心代码**：
```python
def export_selected_users(self, request, queryset):
    """批量导出所选用户"""
    # 创建 CSV 响应
    timestamp = datetime.now().strftime('%Y%m%d_%H%M%S')
    response = HttpResponse(content_type='text/csv; charset=utf-8-sig')
    response['Content-Disposition'] = f'attachment; filename="users_export_{timestamp}.csv"'

    # 添加 BOM 以支持 Excel 正确显示中文
    response.write('﻿')

    writer = csv.writer(response)

    # 写入表头
    writer.writerow([...])

    # 写入数据
    for user in queryset:
        writer.writerow([...])

    # 显示成功消息
    self.message_user(request, f'成功导出 {queryset.count()} 个用户')

    return response

export_selected_users.short_description = '导出所选用户'
```

**关键点**：
1. ✅ 使用 `datetime` 生成时间戳文件名
2. ✅ 设置 `Content-Disposition` 头，触发浏览器下载
3. ✅ 使用 UTF-8-sig 编码，支持 Excel 显示中文
4. ✅ 添加 BOM 标记（`﻿`）
5. ✅ 使用 `get_xxx_display()` 方法显示可读的字段值
6. ✅ 格式化日期时间字段
7. ✅ 显示成功消息

### 2. 注册动作

**代码**：
```python
@admin.register(User)
class UserAdmin(BaseUserAdmin):
    actions = ['export_selected_users']
```

**说明**：
- 将导出方法添加到 `actions` 列表
- Django Admin 会自动在"动作"下拉菜单中显示

## ⚠️ 注意事项

### 1. 数据安全

- **密码不导出**：导出的数据不包含密码字段（加密后的密码）
- **敏感信息**：导出文件包含用户的详细信息，请妥善保管
- **权限控制**：只有管理员可以执行导出操作

### 2. 文件编码

- **UTF-8-sig 编码**：支持 Excel 正确显示中文
- **BOM 标记**：文件开头包含 BOM 标记（`﻿`）
- **兼容性**：兼容 Excel、WPS、Google Sheets 等工具

### 3. 性能考虑

- **大量数据**：导出大量用户时可能需要较长时间
- **建议分批**：如果用户数量超过 10000，建议分批导出
- **服务器负载**：导出操作会占用服务器资源

### 4. 数据完整性

- **实时数据**：导出的是当前时刻的数据
- **筛选条件**：导出的数据受筛选条件影响
- **选择范围**：只导出选中的用户
