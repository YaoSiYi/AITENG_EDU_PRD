## 快速启动

### 方式一: Docker Compose (推荐)

1. **配置环境变量**
```bash
cp .env.example .env
# 编辑 .env 文件,设置数据库密码等
```

2. **启动所有服务**
```bash
docker compose up -d
```

3. **初始化数据库**
```bash
# 运行数据库迁移
docker compose exec backend python manage.py migrate

# 创建超级管理员
docker compose exec backend python manage.py createsuperuser
```

4. **访问系统**
- 前端: http://localhost:3000
- 后台管理: http://localhost:8000/admin
- API 文档: http://localhost:8000/api/docs

### 方式二: 本地开发

#### 后端启动

```bash
cd backend

# 创建虚拟环境
python3 -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate

# 安装依赖
pip install -r requirements.txt

# 配置环境变量
cp ../.env.example .env

# 运行迁移
python manage.py migrate

# 创建超级用户
python manage.py createsuperuser

# 启动开发服务器
python manage.py runserver

# 启动 Celery (另开终端)
celery -A config worker -l info

# 启动 Celery Beat (另开终端)
celery -A config beat -l info
```

#### 前端启动

```bash
cd frontend-web

# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

## 详细功能说明

### 🎯 测试用例管理系统（v0.0.9 重大更新）

#### 1. 智能状态管理
- **已关闭状态**：
  - 整行蓝色高亮背景 `#e6f4ff`
  - 除状态字段外，其他字段自动锁定不可编辑
  - 悬停时背景加深 `#bae0ff`
  
- **已废弃状态**：
  - 整行灰色背景 `#f5f5f5`
  - 文字颜色变灰 `#aaa`
  - 除状态字段外，其他字段自动锁定不可编辑
  - 标签透明度降低
  
- **有效/草稿状态**：
  - 所有字段正常编辑
  - 无特殊样式限制

#### 2. 快速编辑模式
- **列表直接编辑**：
  - 严重程度：下拉选择（高/中/低）
  - 优先级：下拉选择（紧急/高/中/低）
  - 状态：下拉选择（草稿/有效/已废弃/已关闭）
  - 阶段：下拉选择（冒烟测试/预发环境/生产环境/未设置）
  
- **操作特点**：
  - 无需打开编辑弹窗
  - 点击即可修改
  - 实时保存（PATCH 部分更新）
  - 提升操作效率 5 倍

#### 3. 复现步骤编辑器
- **支持内容类型**：
  - 文字描述
  - 图片上传（JPG/PNG/GIF/WEBP/BMP）
  - 混合排列
